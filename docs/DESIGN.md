# Design.md — 日本語UI 設計・表記ルール

## 1. 目的

この文書は以下を定める。

- UIの一貫性を保つ
- 実装とレビューの判断基準にする
- 日本語文言の迷いを減らす

Design.md は「思想」ではなく、**実装時に迷わない粒度の運用ルール集**である。

---

## 2. 対象範囲

- `core/` 配下の全コンポーネント（atoms / molecules / organisms / templates）
- `extensions/` のプロジェクト固有コンポーネントにも原則適用
- Storybook のストーリー内テキスト

---

## 3. デザイン原則

| 原則 | 説明 |
|------|------|
| **明快** | 一読で意味が伝わる。曖昧な表現を避ける |
| **簡潔** | 最小限の文字数で伝える。冗長な敬語を避ける |
| **一貫** | 同じ概念には同じ言葉を使う。表記ゆれを許容しない |
| **安心** | ユーザーが「何が起きたか」「次に何をすればよいか」を常に把握できる |

---

## 4. 日本語UI文言ルール

### 4.1 文体

| 項目 | ルール | 例 |
|------|--------|-----|
| 文体 | **です・ます**（敬体）で統一 | ○ 保存しました / ✕ 保存した |
| ボタン | 動詞で終える（「する」は省略可） | ○ 保存 / ○ 削除する / ✕ 保存です |
| メニュー名 | 名詞 | ○ 設定 / ○ アカウント |
| 説明文 | です・ます | ○ ファイルを選択してください |
| プレースホルダー | 体言止めまたは「〜を入力」 | ○ メールアドレス / ○ 名前を入力 |

### 4.2 表記

| 項目 | ルール | 例 |
|------|--------|-----|
| 日本語と英数字の間 | **半角スペースを入れない** | ○ 全3件 / ✕ 全 3 件 |
| 数字 | 半角 | ○ 3件 / ✕ ３件 |
| カッコ | 全角丸カッコ | ○ （税込） / ✕ (税込) |
| 句読点 | 句点「。」読点「、」 | |
| 長音 | カタカナ語末尾の長音は付ける | ○ サーバー / ✕ サーバ |
| 単位 | 半角英数 | ○ 100MB / ✕ 100ＭＢ |

### 4.3 日付・時刻・通貨

| 項目 | 形式 | 例 |
|------|------|-----|
| 日付（長） | YYYY年M月D日（曜） | 2026年4月13日（月） |
| 日付（短） | YYYY/MM/DD | 2026/04/13 |
| 時刻 | HH:mm（24時間制） | 14:30 |
| 通貨 | ¥{金額}（カンマ区切り） | ¥1,200 |
| 期間 | 〜（波ダッシュ） | 9:00〜17:00 |

### 4.4 日本語組版・フォント

#### フォントスタック

現在の `--font-family-sans` は欧文フォント中心で、日本語フォントの明示指定がない（`system-ui` にフォールバック）。
プロジェクトで日本語フォントを明示する場合は以下を推奨する。

```
/* 推奨順: OS標準の日本語フォントを明示 */
--font-family-sans:
  ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont,
  'Hiragino Sans', 'Hiragino Kaku Gothic ProN',          /* macOS / iOS */
  'Noto Sans JP',                                         /* Android / Web */
  'Yu Gothic UI', 'Yu Gothic', 'Meiryo',                  /* Windows */
  sans-serif,
  'Apple Color Emoji', 'Segoe UI Emoji', 'Noto Color Emoji';
```

**判断基準:**
- システムフォントで十分な場合は現状維持（`system-ui` フォールバック）でよい
- ブランドフォントや Web フォントを使う場合はこの変数を上書きする
- Noto Sans JP 等の Web フォントを使う場合、`font-display: swap` を指定して FOIT（Flash of Invisible Text）を防ぐ

#### 日本語特有のタイポグラフィ問題

| 問題 | 説明 | 対策 |
|------|------|------|
| **文字幅の不均一** | 全角文字は半角の約2倍幅。英数混在で行長がブレる | 固定幅レイアウト（ボタン等）では `min-width` を設定する |
| **行間の狭さ** | 欧文基準の `line-height: 1.5` は日本語では窮屈に見える | 本文は `line-height: 1.75`〜`1.8` を目安にする |
| **禁則処理** | 句読点や閉じカッコが行頭に来る | `word-break: auto-phrase`（Chrome 119+）または `overflow-wrap: anywhere` を使う。`word-break: break-all` は禁則を壊すので避ける |
| **letter-spacing** | 日本語に `letter-spacing` を広げすぎると間延びする | 見出し以外では `letter-spacing: normal` を維持する |
| **font-weight** | 日本語フォントは太字の種類が限られる。`bold`（700）が潰れて見えることがある | 強調は `font-weight: 600`（semibold）を基本とし、700 は大きい見出しに限定する |
| **文字化け** | ファイルの文字コードが UTF-8 以外だと化ける | HTML に `<meta charset="UTF-8">` を必ず含める。ソースファイルも UTF-8 で保存する |

#### 日本語テキストの折り返し

```css
/* 推奨: 自然な日本語の折り返し */
.text-body {
  overflow-wrap: anywhere;
  word-break: normal;       /* break-all は使わない */
  line-break: strict;       /* 厳密な禁則処理 */
}
```

- `word-break: break-all` は句読点が行頭に来るなど禁則違反を起こすため、原則使わない
- テーブルセルなど幅制約がある場所では `overflow: hidden; text-overflow: ellipsis` で省略する

#### 全角・半角の混在

| パターン | ルール | 例 |
|---------|--------|-----|
| 日本語 + 数字 | スペースなし | ○ 全3件 / ✕ 全 3 件 |
| 日本語 + 英単語 | スペースなし | ○ Googleアカウント / ✕ Google アカウント |
| 日本語 + 記号 | 記号は半角 | ○ 100% / ✕ 100％ |
| 全角記号 | カッコのみ全角、他は半角 | ○ （税込）/ ✕ 【税込】 |

#### 入力フィールドの IME 制御

| 入力内容 | `inputmode` | 備考 |
|---------|------------|------|
| 日本語テキスト | （指定なし） | デフォルトで IME ON |
| メールアドレス | `email` | IME 自動 OFF |
| 電話番号 | `tel` | IME 自動 OFF |
| URL | `url` | IME 自動 OFF |
| 数値のみ | `numeric` | IME 自動 OFF |

`ime-mode` は非推奨。`inputmode` 属性で制御する。

## 5. 用語集

同じ概念に複数の言い方が混在することを防ぐ。

| 概念 | 採用語 | 非推奨 | 理由 |
|------|--------|--------|------|
| 認証 | ログイン | サインイン | 既存UIに合わせる |
| 生成 | 作成 | 登録、新規作成 | 動作として明確 |
| 削除 | 削除 | 消去、除去 | 破壊的操作として統一 |
| 変更の確定 | 保存 | 登録、反映、適用 | 最も一般的 |
| 取り消し | キャンセル | 中止、やめる | ボタン文言として統一 |
| 読み込み | 読み込み中 | ローディング | 日本語に統一 |
| 失敗 | 失敗しました | エラーが発生しました | 何が起きたか明示 |
| 検索 | 検索 | サーチ、探す | |
| 編集 | 編集 | 修正、変更 | |
| 戻る | 戻る | 前へ、バック | |
| 次へ | 次へ | 進む、ネクスト | |
| 追加 | 追加 | 登録、挿入 | |

新しい概念が登場したらこの表に追加すること。

## 6. レイアウト原則

### 6.1 余白

CSS変数として定義済みのスペーシングトークンを使う。直値（`12px` 等）は禁止。

| 用途 | トークン目安 |
|------|-------------|
| コンポーネント内部 | `--spacing-1` 〜 `--spacing-3`（0.25rem〜0.75rem） |
| コンポーネント間 | `--spacing-4` 〜 `--spacing-6`（1rem〜1.5rem） |
| セクション間 | `--spacing-8` 〜 `--spacing-12`（2rem〜3rem） |
| ページ余白 | `--spacing-16` 以上（4rem〜） |

### 6.2 レスポンシブ

| ブレークポイント | 用途 |
|----------------|------|
| 〜640px | モバイル |
| 641px〜1024px | タブレット |
| 1025px〜 | デスクトップ |

- モバイルファーストで設計する
- タッチターゲットは最低 44×44px

### 6.3 配置

- 主要アクションは右下または右上に置く
- 破壊的アクション（削除等）は主要アクションから離す
- フォームは上から下へ、1列で並べるのを基本とする

## 7. コンポーネント規約

### 7.1 Button

**用途:** ユーザーが明示的に操作を実行するための部品。

| 種別 | 用途 | 制約 |
|------|------|------|
| Primary | 画面の主要アクション | **1画面に1つまで** |
| Secondary | 補助操作 | |
| Outline | 軽い操作、フィルター等 | |
| Danger | 削除など破壊的操作 | 確認ダイアログ必須 |

**文言ルール:**
- 動詞で終える
- 全角8〜12文字以内を目安
- 対象が曖昧な場合は「○○を削除」のように対象を含める

| 推奨 | 非推奨 | 理由 |
|------|--------|------|
| 保存 | OK | 何をするか不明 |
| 削除する | 実行 | 操作が曖昧 |
| 下書き保存 | 登録完了 | 状態を正確に |
| キャンセル | やめる | 用語集に統一 |

### 7.2 Input

**用途:** ユーザーからテキスト入力を受け付ける。

- ラベルは**常時表示**する（プレースホルダーで代用しない）
- 入力例はプレースホルダーではなく**補助テキスト**で示す
- 必須 / 任意はラベル付近で明示する

| 状態 | 表示 |
|------|------|
| 未入力 | プレースホルダー（薄い文字）|
| 入力中 | ラベル + 入力値 |
| エラー | 赤枠 + エラーメッセージ（入力欄の直下） |
| 無効 | グレーアウト + 操作不可 |

### 7.3 Select / Dropdown

- 選択肢が5個以下ならラジオボタンを検討する
- 未選択状態のラベルは「選択してください」
- 選択肢にはプレースホルダー項目を含めない

### 7.4 Checkbox / Radio

- Checkbox: 複数選択可能な場合
- Radio: 排他選択の場合
- ラベルは必ず付ける（アイコンだけにしない）

### 7.5 Dialog / Modal

| 種別 | 用途 | ボタン構成 |
|------|------|-----------|
| Alert | 通知（情報提供のみ） | 閉じる |
| Confirm | 確認（ユーザーに判断を求める） | キャンセル / [操作名] |
| Form | 入力を伴う操作 | キャンセル / 保存 |

- モーダルタイトルは全角20文字以内
- 閉じる手段は必ず用意する（×ボタン or Escape）
- 破壊的操作の確認では「本当に○○しますか？」ではなく、**結果を明示**する

```
✕ 本当に削除しますか？
○ このファイルを削除しますか？削除したファイルは元に戻せません。
```

### 7.6 Toast / Banner

| 種別 | 用途 | 自動消去 |
|------|------|---------|
| Success | 操作成功 | あり（3〜5秒） |
| Warning | 注意喚起 | なし |
| Error | エラー通知 | なし |
| Info | 情報提供 | あり（5秒） |

- エラーと警告は自動消去しない（ユーザーが見逃す）
- メッセージは1行に収める

### 7.7 Table / DataTable

- 列見出しは省略語を使う場合、用語集に登録する
- 空状態は EmptyState コンポーネントで対応する
- ソート可能な列はアイコンで示す

### 7.8 Tabs / TabBar

- タブ名は全角6文字以内を推奨
- アクティブタブは視覚的に明確に区別する
- タブの数は5個以内を推奨（超える場合はナビゲーション再設計を検討）

### 7.9 FormField

- ラベル + 入力 + 補助テキスト + エラーメッセージの4要素をセットで管理
- ラベルを省略しない
- 必須項目は「必須」バッジで示す（`*` のみは不可）

## 8. 状態設計

すべてのコンポーネントについて、以下の状態を考慮すること。

| 状態 | 説明 | 対応 |
|------|------|------|
| **初期** | 初回表示、データ未取得 | 適切な初期値またはスケルトン |
| **読み込み中** | データ取得中 | Spinner または Skeleton |
| **通常** | データ表示中 | 正常表示 |
| **入力中** | ユーザーが操作中 | フォーカス状態の視覚表示 |
| **完了** | 操作成功 | Success Toast |
| **エラー** | 操作失敗 or バリデーションエラー | エラーメッセージ表示 |
| **空** | データが0件 | EmptyState（次のアクションを提示） |
| **無効** | 操作不可 | グレーアウト + 理由の表示 |
| **権限不足** | 権限がない | メッセージ + 代替手段の提示 |

### 空状態のルール

空状態は「何もない」だけでなく、**次に何をすればよいか**を示す。

```
✕ データがありません
○ まだプロジェクトがありません。「作成」ボタンから最初のプロジェクトを追加しましょう。
```

## 9. エラーメッセージ / 確認文

### 9.1 エラーメッセージ

以下の順序で構成する。

1. **何が起きたか**（状況）
2. **何をすればよいか**（対処）
3. 必要なら原因

| 推奨 | 非推奨 | 理由 |
|------|--------|------|
| メールアドレスを入力してください。 | 入力エラー | 何を直せばよいか不明 |
| パスワードは8文字以上で入力してください。 | 不正な値です | 条件が不明 |
| 保存に失敗しました。時間をおいて再度お試しください。 | エラーが発生しました | 原因も対処も不明 |
| ファイルサイズが上限（10MB）を超えています。 | 無効なファイルです | 何が問題か不明 |

**テンプレート:**

```
[バリデーション] {項目名}を{条件}で入力してください。
[操作失敗]     {操作}に失敗しました。{対処方法}。
[権限不足]     {操作}する権限がありません。管理者にお問い合わせください。
[上限超過]     {項目}が上限（{上限値}）を超えています。
```

### 9.2 確認ダイアログ

**構成:**
- タイトル: 操作名（「削除の確認」等）
- 本文: 何が起きるかを具体的に記述
- 取り消し不可なら必ず明記
- ボタン: キャンセル / [具体的な操作名]

```
タイトル: ファイルの削除
本文:    「報告書_2026.pdf」を削除しますか？
         削除したファイルは元に戻せません。
ボタン:  [キャンセル] [削除する]
```

**ボタン文言のルール:**
- 「はい / いいえ」は使わない
- 操作を具体的に書く（「削除する」「送信する」）
- 破壊的操作のボタンは Danger スタイルを使う

## 10. アクセシビリティ

### 10.1 必須要件

| 項目 | ルール |
|------|--------|
| 色 | 色だけで意味を伝えない（アイコン・テキストを併用） |
| コントラスト | テキストは WCAG AA 以上（4.5:1） |
| フォーカス | キーボードフォーカスが視認できること |
| ラベル | フォーム要素には必ずラベルを関連付ける |
| alt | 意味のある画像には代替テキストを付ける |
| aria | 動的な状態変化は `aria-live` で通知する |
| 必須表示 | 色（赤い `*`）だけでなく「必須」テキストで示す |

### 10.2 エラー表示

- エラーは入力欄の**直下**に表示する（ページ上部にまとめない）
- エラー発生時、該当の入力欄にフォーカスを移動する
- エラーメッセージと入力欄を `aria-describedby` で関連付ける

### 10.3 スクリーンリーダー

- アイコンボタンには `aria-label` を付ける
- 装飾目的の画像は `aria-hidden="true"`
- モーダル表示中は背景を `aria-hidden="true"` にする

## 11. 禁止事項

以下はレビューで指摘・差し戻しの対象とする。

| 禁止 | 理由 |
|------|------|
| 「OK」「実行」など意味が曖昧なボタン文言 | ユーザーが結果を予測できない |
| プレースホルダーだけで入力内容を説明 | 入力中に消えて確認できない |
| 同一画面で「保存」と「登録」が混在 | 表記ゆれ |
| 1画面に Primary Button を2つ以上 | 主要アクションが不明になる |
| 「エラーが発生しました」だけの表示 | 原因も対処も伝わらない |
| `*` だけで必須を示す | アクセシビリティ違反 |
| 色だけで状態を区別（赤=エラー等） | 色覚多様性への配慮不足 |
| 確認ダイアログのボタンが「はい / いいえ」 | 操作が曖昧 |
| ラベルなしのアイコンボタン | スクリーンリーダーで読めない |
| 自動消去されるエラーメッセージ | ユーザーが見逃す |

## 12. 例外ルール

以下は原則の例外として許容する。

| 例外 | 条件 |
|------|------|
| ボタン文言の文字数超過 | 意味の明確さが優先される場合（レビューで承認） |
| Primary Button の複数配置 | ウィザード形式で「戻る」「次へ」が並ぶ場合 |
| 英語表記 | 技術用語で日本語化すると意味が変わる場合（例: API, URL） |
| プレースホルダーによる入力例 | 検索バーなど、ラベルが視覚的に自明な場合 |

例外を適用した場合は、コードレビューで理由を明記すること。

## 13. 文字数ガイドライン

| 要素 | 目安 | 超過時 |
|------|------|--------|
| Primary Button | 全角4〜8文字 | 文言を短縮 |
| Secondary Button | 全角4〜12文字 | 文言を短縮 |
| タブ名 | 全角6文字以内 | 省略語を用語集に登録 |
| モーダルタイトル | 全角20文字以内 | 説明は本文に逃がす |
| テーブル列見出し | 全角8文字以内 | 省略語を用語集に登録 |
| Toast メッセージ | 1行（全角30文字程度） | 詳細はリンクで誘導 |
| エラーメッセージ | 2行以内 | 詳細は補助テキストで |
